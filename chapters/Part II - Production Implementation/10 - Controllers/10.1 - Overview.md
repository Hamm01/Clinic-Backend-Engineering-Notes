# What Is a Controller?

A Controller is the first layer of our business logic that receives an HTTP request after it has passed through all middleware.

Its primary responsibility is to:

- Receive the request.
- Read request data.
- Call the appropriate Service.
- Return an HTTP response.

A Controller should **coordinate** work, not perform business logic.

---

# Responsibilities of a Controller

A production-ready Controller should:

- Accept incoming HTTP requests.
- Read URL parameters.
- Read query parameters.
- Read request body.
- Read authenticated user information.
- Call the correct Service.
- Return standardized HTTP responses.

A Controller should **not**:

- Write SQL queries.
- Access Prisma directly.
- Implement business rules.
- Perform authorization checks.
- Contain complex calculations.

Those responsibilities belong elsewhere in the application.
