# Step 1 ‚Äî Create Controller

```
doctor.controller.ts
```

```ts
import { Request, Response } from 'express'

import { DoctorService } from './doctor.service'

export class DoctorController {
  private readonly doctorService = new DoctorService()

  async create(req: Request, res: Response) {
    const doctor = await this.doctorService.create({
      ...req.body,

      clinicId: req.user.clinicId,

      createdBy: req.user.id
    })

    return res.status(201).json({
      success: true,

      message: 'Doctor created successfully.',

      data: doctor
    })
  }
}
```

# Understanding the Controller

The Controller has only three responsibilities.

## 1. Read Request

```ts
req.body
```

Contains

```
Doctor Information
```

---

## 2. Call Service

```ts
doctorService.create(...)
```

Business logic is delegated.

The Controller never creates the Doctor itself.

---

## 3. Return Response

```ts
return res.status(201).json(...)
```

The Controller sends the final response to the client.

# Why Add clinicId Here?

Notice

```ts
clinicId: req.user.clinicId
```

instead of

```ts
req.body.clinicId
```

Reason

The authenticated user belongs to a clinic.

The client should never choose

```
clinicId
```

Otherwise someone could create Doctors for another clinic.

Always trust

```
JWT

‚Üì

req.user
```

Never trust

```
Client Input
```

---

# Why Add createdBy?

Every record should know

who created it.

```ts
createdBy: req.user.id
```

Later we can answer

```
Who created this Doctor?

When?

From which account?
```

This becomes extremely useful for

- Audit Logs
- Debugging
- Compliance
- Healthcare Regulations

# Production Notes

Always

- Keep Controllers small.
- Delegate business logic.
- Use authenticated user information.
- Return standardized responses.
- Never trust client-controlled ownership fields.

---

# Common Mistakes

## Writing Business Logic

Wrong

```ts
if(
doctor.experience > 10
)
```

Move this to the Service.

---

## Calling Prisma

Wrong

```ts
await prisma.doctor.create(...)
```

Controllers should never access the database directly.

---

## Trusting Client Ownership

Wrong

```ts
clinicId: req.body.clinicId
```

Correct

```ts
clinicId: req.user.clinicId
```

---

## Different Response Formats

Every Controller should return the same response structure.

Consistency simplifies frontend integration.
