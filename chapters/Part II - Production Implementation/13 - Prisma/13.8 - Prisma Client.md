# 13.8 - Prisma Client

## Overview

The Prisma Client is a type-safe database client generated from the `schema.prisma` file.

It provides a TypeScript API for interacting with database models without writing SQL manually.

Every time the schema changes, Prisma regenerates the client to reflect the latest models.

---

## Generating Prisma Client

Generate the Prisma Client using:

```bash
npx prisma generate
```

Prisma reads the `schema.prisma` file and creates a type-safe client inside `node_modules`.

---

## Importing Prisma Client

Import the generated client.

```ts
import { PrismaClient } from '@prisma/client'
```

Create a single instance.

```ts
const prisma = new PrismaClient()
```

---

## Creating a Shared Prisma Client

In our project, we create only one Prisma Client instance.

```text
src/

└── database/

    └── prisma.ts
```

```ts
import { PrismaClient } from '@prisma/client'

export const prisma = new PrismaClient()
```

Every Repository imports this instance.

---

## Using Prisma Client

Every model defined inside `schema.prisma` becomes available on the Prisma Client.

Example

```prisma
model Patient {

    id   String @id @default(cuid())

    name String

}
```

Prisma generates

```ts
prisma.patient
```

Likewise,

```prisma
model Doctor
```

becomes

```ts
prisma.doctor
```

Every model has its own query API.

---

## Common Operations

Example

```ts
prisma.patient.findUnique()

prisma.patient.findMany()

prisma.patient.create()

prisma.patient.update()

prisma.patient.delete()
```

These methods are generated automatically.

---

## Implementation In Our Project

Example

```ts
import { prisma } from '../database/prisma'

export class PatientRepository {
  async findById(id: string) {
    return prisma.patient.findUnique({
      where: {
        id
      }
    })
  }
}
```

Notice that the Repository communicates directly with Prisma.

The Service communicates only with the Repository.

---

## Request Flow

```text
Controller

↓

Service

↓

PatientRepository

↓

Prisma Client

↓

PostgreSQL
```

Prisma should never be accessed directly from the Controller or Service.

---

## Type Safety

One of Prisma's biggest advantages is automatic type safety.

Example

```ts
const patient = await prisma.patient.findUnique({
  where: {
    id: '123'
  }
})
```

The returned object is automatically typed.

This provides:

- Auto-completion
- Compile-time validation
- Reduced runtime errors

---

## Regenerating the Client

Whenever `schema.prisma` changes, regenerate the client.

```bash
npx prisma generate
```

Without regeneration, newly added models and fields will not be available.

---

## Best Practices

- Create only one Prisma Client instance.
- Share the client across the application.
- Use Prisma only inside Repositories.
- Regenerate the client after schema changes.
- Allow TypeScript to infer types whenever possible.

---

## Common Mistakes

❌ Creating multiple `PrismaClient` instances.

❌ Using Prisma inside Controllers.

❌ Forgetting to regenerate the client.

❌ Editing generated Prisma files.

---

## Summary

The Prisma Client is the API our application uses to communicate with the database.

It is generated automatically from `schema.prisma` and provides a fully type-safe interface for querying and modifying data.

In our Clinic Backend, every Repository uses this shared Prisma Client to perform database operations.
