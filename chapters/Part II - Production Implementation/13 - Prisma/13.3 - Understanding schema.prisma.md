# 13.3 - Understanding schema.prisma

## Overview

The `schema.prisma` file is the heart of every Prisma project.

It defines:

- Database connection
- Prisma Client generation
- Database models
- Relationships

Whenever the schema changes, Prisma uses it to generate a new Prisma Client and create database migrations.

---

## Location

The schema is located inside the `prisma` folder.

```text
project/

├── prisma/
│   └── schema.prisma
│
├── src/
│
└── package.json
```

---

## Structure

A typical Prisma schema contains three sections.

```prisma
generator client {

  provider = "prisma-client-js"

}

datasource db {

  provider = "postgresql"

  url = env("DATABASE_URL")

}

model Patient {

}
```

---

## Generator

The generator tells Prisma what type of client should be created.

```prisma
generator client {

  provider = "prisma-client-js"

}
```

When running

```bash
npx prisma generate
```

Prisma generates a fully type-safe TypeScript client.

---

## Datasource

The datasource tells Prisma which database to connect to.

```prisma
datasource db {

  provider = "postgresql"

  url = env("DATABASE_URL")

}
```

Common providers include:

- PostgreSQL
- MySQL
- SQLite
- SQL Server
- MongoDB

The connection string is read from the `.env` file.

---

## Models

Models define the database tables.

Example

```prisma
model Patient {

  id    String @id @default(cuid())

  name  String

  email String @unique

}
```

Each model becomes a table in the database.

---

## How Prisma Uses schema.prisma

```
schema.prisma

↓

Prisma CLI

↓

Generate Prisma Client

↓

Application
```

Whenever a model changes:

```bash
npx prisma generate
```

should be executed.

## Implementation In Our Project

Current folder structure:

```text
prisma/

└── schema.prisma

src/

├── database/

│   └── prisma.ts

└── repositories/
```

Repositories use the generated Prisma Client.

They never read `schema.prisma` directly.

---

## Best Practices

- Keep all models inside `schema.prisma`.
- Use meaningful model names.
- Keep datasource configuration in one place.
- Generate the Prisma Client after every schema change.
- Store database credentials inside `.env`.

---

## Common Mistakes

❌ Editing generated Prisma Client files.

❌ Forgetting to regenerate the client.

❌ Hardcoding database URLs.

❌ Defining duplicate models.

---

## Summary

The `schema.prisma` file is the source of truth for the Prisma project.

It defines how Prisma connects to the database, what models exist, and how the Prisma Client is generated.

Every schema change starts here.
