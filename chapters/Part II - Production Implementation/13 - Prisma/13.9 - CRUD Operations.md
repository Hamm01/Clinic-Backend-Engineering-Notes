# 13.9 - CRUD Operations

## Overview

CRUD stands for:

- Create
- Read
- Update
- Delete

These are the four basic operations performed on database records.

Prisma provides dedicated methods for each operation through the generated Prisma Client.

---

## Create

Create inserts a new record into the database.

Example

```ts
const patient = await prisma.patient.create({
  data: {
    firstName: 'John',

    lastName: 'Doe',

    email: 'john@example.com'
  }
})
```

The returned object contains the newly created record.

---

## Read One Record

Retrieve a single record using `findUnique()`.

Example

```ts
const patient = await prisma.patient.findUnique({
  where: {
    id: patientId
  }
})
```

Returns:

- The matching record
- `null` if no record exists

---

## Read Multiple Records

Retrieve multiple records using `findMany()`.

Example

```ts
const patients = await prisma.patient.findMany()
```

Returns an array of records.

```ts
[
    { ... },

    { ... },

    { ... }
]
```

---

## Update

Update modifies an existing record.

Example

```ts
const patient = await prisma.patient.update({
  where: {
    id: patientId
  },

  data: {
    phone: '9876543210'
  }
})
```

Only the specified fields are updated.

---

## Delete

Delete removes a record.

Example

```ts
await prisma.patient.delete({
  where: {
    id: patientId
  }
})
```

The record is permanently removed from the database.

---

## CRUD Flow

```
Create

↓

Read

↓

Update

↓

Delete
```

These operations form the foundation of most backend APIs.

---

## Implementation In Our Project

Patient Repository

```ts
import { prisma } from '../database/prisma'

export class PatientRepository {
  async create(data: CreatePatientDto) {
    return prisma.patient.create({
      data
    })
  }

  async findById(id: string) {
    return prisma.patient.findUnique({
      where: {
        id
      }
    })
  }

  async update(
    id: string,

    data: UpdatePatientDto
  ) {
    return prisma.patient.update({
      where: {
        id
      },

      data
    })
  }

  async delete(id: string) {
    return prisma.patient.delete({
      where: {
        id
      }
    })
  }
}
```

The Repository exposes business-oriented methods while Prisma performs the database operations.

---

## Error Handling

Some operations throw exceptions.

Example

```ts
await prisma.patient.update({
  where: {
    id: 'invalid-id'
  },

  data: {
    firstName: 'Ronak'
  }
})
```

If the record does not exist,

Prisma throws an error.

Repositories should handle or propagate these errors appropriately.

---

## Best Practices

- Use `create()` for inserting records.
- Use `findUnique()` when querying by unique fields.
- Use `findMany()` for collections.
- Update only the fields that change.
- Delete records carefully.

---

## Common Mistakes

❌ Using `findMany()` when expecting one record.

❌ Updating records without checking existence.

❌ Deleting records accidentally.

❌ Putting business logic inside CRUD methods.

❌ Calling Prisma directly from Services.

---

## Summary

CRUD operations are the most common database interactions in any backend application.

Prisma provides a simple, type-safe API for creating, reading, updating, and deleting records, while Repositories encapsulate these operations within the application's architecture.
