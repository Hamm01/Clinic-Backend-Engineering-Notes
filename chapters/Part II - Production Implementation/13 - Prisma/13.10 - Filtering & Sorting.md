# 13.10 - Filtering & Sorting

## Overview

Retrieving all records from a database is rarely sufficient for real-world applications.

Most APIs need to:

- Find a specific set of records.
- Filter based on conditions.
- Sort the results.
- Limit the returned data.

Prisma provides built-in support for filtering and sorting through the generated Prisma Client.

---

## Filtering Records

Use the `where` clause to filter records.

Example

```ts
const patients = await prisma.patient.findMany({
  where: {
    isActive: true
  }
})
```

Only active patients are returned.

---

## Filtering by Multiple Fields

Multiple conditions can be combined.

Example

```ts
const patients = await prisma.patient.findMany({
  where: {
    firstName: 'John',

    isActive: true
  }
})
```

Both conditions must match.

---

## Comparison Operators

Prisma supports comparison operators.

Example

```ts
const patients = await prisma.patient.findMany({
  where: {
    age: {
      gte: 18
    }
  }
})
```

Common operators include:

- `equals`
- `gt`
- `gte`
- `lt`
- `lte`
- `not`

---

## String Filtering

Search using string operators.

Example

```ts
const doctors = await prisma.doctor.findMany({
  where: {
    name: {
      contains: 'John'
    }
  }
})
```

Useful operators:

- `contains`
- `startsWith`
- `endsWith`

---

## Logical Operators

Combine multiple conditions.

Example

```ts
const patients = await prisma.patient.findMany({
  where: {
    AND: [
      {
        isActive: true
      },

      {
        age: {
          gte: 18
        }
      }
    ]
  }
})
```

Prisma also supports:

- `OR`
- `NOT`

---

## Sorting Results

Sort records using `orderBy`.

Example

```ts
const patients = await prisma.patient.findMany({
  orderBy: {
    createdAt: 'desc'
  }
})
```

Ascending order

```ts
createdAt: 'asc'
```

Descending order

```ts
createdAt: 'desc'
```

---

## Selecting Specific Fields

Retrieve only the required fields.

Example

```ts
const patients = await prisma.patient.findMany({
  select: {
    id: true,

    firstName: true,

    email: true
  }
})
```

This reduces the amount of data returned.

---

## Including Related Data

Load related models using `include`.

Example

```ts
const appointments = await prisma.appointment.findMany({
  include: {
    doctor: true,

    patient: true
  }
})
```

Related records are fetched automatically.

---

## Implementation In Our Project

Patient Repository

```ts
async findActivePatients() {

    return prisma.patient.findMany({

        where: {

            isActive: true

        },

        orderBy: {

            createdAt: "desc"

        }

    });

}
```

Doctor Repository

```ts
async searchByName(name: string) {

    return prisma.doctor.findMany({

        where: {

            name: {

                contains: name

            }

        }

    });

}
```

---

## Best Practices

- Filter as early as possible.
- Select only required fields.
- Use sorting for predictable results.
- Avoid returning unnecessary data.
- Keep filtering logic inside Repositories.

---

## Common Mistakes

❌ Returning every column.

❌ Forgetting to sort results.

❌ Loading unnecessary relationships.

❌ Building filtering logic inside Controllers.

❌ Fetching all records when filtering is possible.

---

## Summary

Filtering and sorting allow applications to retrieve only the required data.

Prisma provides a clean and type-safe API for applying conditions, ordering results, selecting fields, and loading related records.

These features help build efficient and maintainable database queries.
