```text

                          ▼
```

## Database Layer

The shared Prisma Client is created once.

```text
src/

database/

└── prisma.ts
```

```ts
import { PrismaClient } from '@prisma/client'

export const prisma = new PrismaClient()
```

Every Repository imports this instance.

---

## Repository Layer

Repositories communicate with Prisma.

Example

```ts
import { prisma } from '../database/prisma'

export class PatientRepository {
  async findById(id: string) {
    return prisma.patient.findUnique({
      where: {
        id
      }
    })
  }
}
```

Repositories contain all database queries.

---

## Service Layer

Services implement business logic.

```ts
export class PatientService {
  constructor(private readonly patientRepository: PatientRepository) {}

  async getPatient(id: string) {
    return this.patientRepository.findById(id)
  }
}
```

Services never communicate directly with Prisma.

---

## Controller Layer

Controllers handle HTTP communication.

```ts
@Get(":id")
async getPatient(

    @Param("id") id: string

) {

    return this.patientService.getPatient(id);

}
```

Controllers know nothing about the database.

Every layer performs one responsibility.

---

## Key Principles

Throughout our Clinic Backend, we follow these principles:

- One shared Prisma Client.
- One Repository per entity.
- Prisma used only inside Repositories.
- Services contain business logic.
- Controllers handle HTTP requests.
- Migrations manage schema changes.
- Seed scripts prepare development data.

These principles keep the project modular and maintainable.
