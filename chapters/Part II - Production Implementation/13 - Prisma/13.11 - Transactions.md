# 13.11 - Transactions

## Overview

A transaction allows multiple database operations to execute as a single unit of work.

If every operation succeeds, the transaction is committed.

If any operation fails, all previous changes are rolled back.

Transactions help maintain data consistency.

---

## Why Use Transactions?

Consider booking an appointment.

The application needs to:

- Create an appointment.
- Create a diagnosis record.
- Update the doctor's availability.

If one operation fails, the database should return to its previous state.

This is achieved using a transaction.

---

## Prisma Transactions

Prisma provides the `$transaction()` method.

```ts
await prisma.$transaction([
  prisma.patient.create({
    data: patientData
  }),
  prisma.appointment.create({
    data: appointmentData
  })
])
```

If either query fails, neither record is saved.

---

## Interactive Transactions

Use the callback syntax when operations depend on previous results.

```ts
await prisma.$transaction(async tx => {
  const patient = await tx.patient.create({
    data: patientData
  })

  await tx.appointment.create({
    data: {
      patientId: patient.id,

      doctorId
    }
  })
})
```

The transaction is committed only if every operation succeeds.

---

## Implementation In Our Project

Appointment Service

```ts
await prisma.$transaction(async tx => {
  const appointment = await tx.appointment.create({
    data: appointmentData
  })

  await tx.diagnosis.create({
    data: {
      appointmentId: appointment.id,

      notes
    }
  })
})
```

Both records are created together.


The transaction ensures all operations succeed together.

---

## When to Use Transactions

Use transactions when:

- Creating related records.
- Updating multiple tables.
- Transferring data.
- Maintaining consistency across models.

Avoid transactions for a single database operation.

---

## Best Practices

- Keep transactions short.
- Include only required queries.
- Handle errors appropriately.
- Avoid long-running operations inside a transaction.
- Use interactive transactions when later queries depend on earlier results.

---

## Common Mistakes

❌ Wrapping every query inside a transaction.

❌ Making external API calls inside a transaction.

❌ Performing slow operations before committing.

❌ Ignoring transaction failures.

---

## Summary

Transactions ensure that multiple database operations either succeed together or fail together.

Prisma provides a simple and type-safe API for implementing transactions while maintaining data consistency in production applications.
