# 13.12 - Seeding

## Overview

Seeding is the process of inserting initial data into the database.

Instead of manually creating records every time the database is reset, Prisma allows us to automate this process using a seed script.

Seeding is commonly used during development, testing, and local setup.

---

## Why Seed a Database?

Imagine starting the Clinic Backend with an empty database.

There would be:

- No clinics
- No doctors
- No patients
- No appointments

Before testing the application, we would need to create all this data manually.

A seed script automates this process.

---

## Create a Seed File

Create a seed file inside the `prisma` folder.

```text
prisma/

├── schema.prisma

├── seed.ts

└── migrations/
```

---

## Basic Seed Script

```ts
import { PrismaClient } from '@prisma/client'

const prisma = new PrismaClient()

async function main() {
  await prisma.clinic.create({
    data: {
      name: 'City Health Clinic',

      address: 'Sector 21'
    }
  })
}

main()
  .catch(console.error)
  .finally(async () => {
    await prisma.$disconnect()
  })
```

This inserts one clinic into the database.

---

## Configure package.json

Tell Prisma where the seed file is located.

```json
{
  "prisma": {
    "seed": "tsx prisma/seed.ts"
  }
}
```

---

## Run the Seed Script

Execute the seed command.

```bash
npx prisma db seed
```

Prisma runs the configured seed file and inserts the data.

---

## Seeding in Our Project

For the Clinic Backend, the seed script may create:

- Default Clinic
- Admin User
- Doctors
- Patients
- Specializations

Example

```text
Clinic

↓

Doctors

↓

Patients

↓

Appointments
```

This gives developers a working environment immediately after setup.

---

## Seed After Reset

When resetting the database:

```bash
npx prisma migrate reset
```

Prisma automatically:

- Drops the database
- Applies migrations
- Executes the seed script

The application is ready for development again.

---

## Best Practices

- Seed only development or testing data.
- Keep seed scripts idempotent when possible.
- Use realistic sample data.
- Keep the seed file organized.
- Disconnect Prisma after execution.

---

## Common Mistakes

❌ Seeding production data.

❌ Creating duplicate records on every run.

❌ Forgetting to disconnect Prisma.

❌ Mixing test logic with seed logic.

---

## Summary

Seeding automates the creation of initial database records.

It allows developers to start working with a populated database immediately, making development and testing faster and more consistent.
