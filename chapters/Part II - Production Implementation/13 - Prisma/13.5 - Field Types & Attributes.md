# 13.5 - Field Types & Attributes

## Overview

Fields define the information stored inside a model.

Every field has:

- A name
- A data type
- Zero or more attributes

Attributes provide additional information such as primary keys, uniqueness, default values, and timestamps.

---

## Basic Syntax

```prisma
model Patient {

    id    String

    name  String

    email String

}
```

Each field consists of:

```
<Field Name> <Data Type> <Attributes>
```

Example

```prisma
id String @id
```

---

## Common Data Types

Prisma provides several built-in data types.

| Type       | Description            |
| ---------- | ---------------------- |
| `String`   | Text values            |
| `Int`      | Integer numbers        |
| `Float`    | Decimal numbers        |
| `Boolean`  | True or False          |
| `DateTime` | Date and Time          |
| `Json`     | JSON Object            |
| `Bytes`    | Binary data            |
| `Decimal`  | High precision decimal |

Example

```prisma
model Patient {

    id        String

    age       Int

    weight    Float

    isActive  Boolean

    createdAt DateTime

}
```

---

## Primary Key

Every model should have one primary key.

```prisma
id String @id
```

Usually we generate it automatically.

```prisma
id String @id @default(cuid())
```

Other common options include:

```prisma
@default(uuid())
```

or

```prisma
@default(autoincrement())
```

depending on the database.

---

## Default Values

Default values are assigned automatically.

Example

```prisma
isActive Boolean @default(true)
```

Another example

```prisma
createdAt DateTime @default(now())
```

No value needs to be supplied during insertion.

---

## Unique Fields

A unique field cannot contain duplicate values.

Example

```prisma
email String @unique
```

Valid

```
abc@gmail.com

xyz@gmail.com
```

Invalid

```
abc@gmail.com

abc@gmail.com
```

Prisma and the database both enforce uniqueness.

---

## Optional Fields

A field becomes optional by adding `?`.

```prisma
middleName String?
```

This field may contain `NULL`.

Required field

```prisma
name String
```

Optional field

```prisma
profileImage String?
```

---

## Updated Timestamp

Prisma can automatically update timestamps.

```prisma
updatedAt DateTime @updatedAt
```

Every update automatically changes this value.

---

## Complete Example

```prisma
model Patient {

    id          String   @id @default(cuid())

    firstName   String

    lastName    String

    email       String   @unique

    phone       String

    age         Int?

    isActive    Boolean  @default(true)

    createdAt   DateTime @default(now())

    updatedAt   DateTime @updatedAt

}
```

This model is ready for production and follows common Prisma conventions.

---

## Implementation In Our Project

Most models in the Clinic Backend follow this pattern.

Example

```prisma
model Doctor {

    id        String   @id @default(cuid())

    name      String

    email     String   @unique

    createdAt DateTime @default(now())

    updatedAt DateTime @updatedAt

}
```

This provides:

- Automatic IDs
- Unique emails
- Creation timestamps
- Update timestamps

---

## Best Practices

- Always define a primary key.
- Use `@unique` only when necessary.
- Use `@default(now())` for creation timestamps.
- Use `@updatedAt` for modification timestamps.
- Make fields optional only when business rules allow.

---

## Common Mistakes

❌ Forgetting a primary key.

❌ Making too many fields optional.

❌ Using `@unique` unnecessarily.

❌ Setting timestamps manually.

❌ Using the wrong data type.

---

## Summary

Field types determine what data can be stored,

while attributes define how the database manages those fields.

Choosing appropriate types and attributes results in a reliable, maintainable, and production-ready database schema.
