# 13.1 - Prisma Overview

## Overview

Prisma is a modern ORM (Object Relational Mapper) for Node.js and TypeScript.

It allows us to communicate with relational databases using TypeScript instead of writing SQL for every operation.

Prisma automatically generates a type-safe client based on our database schema.

---

## Why Prisma Exists

Without Prisma, every database operation requires writing SQL manually.

Example:

```sql
SELECT * FROM Patient WHERE id = 1;
```

Using Prisma:

```ts
const patient = await prisma.patient.findUnique({
  where: {
    id: 1
  }
})
```

Prisma generates the SQL internally.

---

## Where Prisma Is Used In Our Project

In our Clinic Backend, Prisma is only used inside the Repository layer.

```text
Controller

↓

Service

↓

Repository

↓

Prisma Client

↓

PostgreSQL
```

Controllers and Services never access Prisma directly.

## Implementation In Our Project

We create a single Prisma Client instance.

```text
src/

database/

└── prisma.ts
```

```ts
import { PrismaClient } from '@prisma/client'

export const prisma = new PrismaClient()
```

Every Repository imports this instance.

Example:

```ts
import { prisma } from '../database/prisma'

export class PatientRepository {
  async findById(id: string) {
    return prisma.patient.findUnique({
      where: {
        id
      }
    })
  }
}
```

---

## Best Practices

- Create only one Prisma Client instance.
- Use Prisma only inside Repositories.
- Never access Prisma from Controllers.
- Never access Prisma from Services.
- Keep queries inside Repository methods.

---

## Common Mistakes

❌ Creating multiple Prisma Clients.

❌ Using Prisma inside Controllers.

❌ Mixing business logic with database queries.

---

## Summary

Prisma is the bridge between our Repository layer and PostgreSQL.

In our project, every database query passes through the Repository before reaching Prisma.
