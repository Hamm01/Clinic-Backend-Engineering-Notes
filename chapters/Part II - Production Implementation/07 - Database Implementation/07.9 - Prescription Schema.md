# 07.9 - Prescription Schema

> A Prescription represents the medicines and treatment plan prescribed by a doctor after making a diagnosis.

---

# Objective

In this chapter we will:

- Create the Prescription entity.
- Associate prescriptions with diagnoses.
- Store medicine information.
- Support multiple medicines for a single diagnosis.

---

# Business Requirement

After diagnosing a patient, the doctor prescribes treatment.

A prescription should contain:

- Medicine Name
- Dosage
- Frequency
- Duration
- Instructions

A patient may receive multiple medicines for one diagnosis.

Therefore,

one Diagnosis can have many Prescriptions.

---

# Prisma Model

Open

```text
prisma/schema.prisma
```

Add

```prisma
model Prescription {

  id            String   @id @default(uuid())

  diagnosisId   String

  medicineName  String

  dosage        String

  frequency     String

  duration      String

  instructions  String?

  createdAt     DateTime @default(now())

  updatedAt     DateTime @updatedAt

  diagnosis Diagnosis
      @relation(fields: [diagnosisId], references: [id])
}
```

# Relationship

```

      │

      ▼

```

# PostgreSQL Representation

Conceptually

```sql
CREATE TABLE prescriptions (

    id UUID PRIMARY KEY,

    diagnosis_id UUID NOT NULL,

    medicine_name TEXT NOT NULL,

    dosage TEXT NOT NULL,

    frequency TEXT NOT NULL,

    duration TEXT NOT NULL,

    instructions TEXT,

    created_at TIMESTAMP,

    updated_at TIMESTAMP,

    FOREIGN KEY (diagnosis_id)

        REFERENCES diagnoses(id)

);
```

# Why This Design?

Medicines are independent treatment items.

A diagnosis may require:

- One medicine
- Two medicines
- Five medicines

Keeping medicines in their own table provides flexibility.

It also allows future features such as:

- Drug interaction checks
- Inventory integration
- Pharmacy systems
- Electronic prescriptions

# Common Mistakes

### Saving Medicines Inside Diagnosis

Wrong

```
Diagnosis

medicine1

medicine2

medicine3
```

The number of medicines varies.

They deserve their own entity.

---

### Combining Dosage and Medicine

Wrong

```
Paracetamol 500mg Twice Daily
```

Store each attribute separately.

This makes searching and reporting easier.

---

### Copying Patient Information

Prescription already reaches the patient through:

```
Prescription

↓

Diagnosis

↓

Medical Record

↓

Appointment

↓

Patient
```

Do not duplicate patient details.
