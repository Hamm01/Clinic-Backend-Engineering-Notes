# 07.8 - Diagnosis Schema

> A diagnosis represents the doctor's medical conclusion after examining the patient during a consultation.

---

# Objective

In this chapter we will:

- Create the Diagnosis entity.
- Associate diagnoses with a medical record.
- Store the doctor's conclusions.
- Support multiple diagnoses for a single consultation.

---

# Business Requirement

After examining a patient, the doctor identifies one or more medical conditions.

Examples:

- Viral Fever
- Hypertension
- Diabetes
- Migraine
- Seasonal Allergy

These conclusions should be stored separately from the consultation notes.

A consultation may produce multiple diagnoses.

Therefore,

one Medical Record can contain many Diagnoses.

---

# Prisma Model

Open

```text
prisma/schema.prisma
```

Add

```prisma
model Diagnosis {

  id               String   @id @default(uuid())

  medicalRecordId  String

  diagnosisName    String

  description      String?

  severity         Severity

  createdAt        DateTime @default(now())

  updatedAt        DateTime @updatedAt

  medicalRecord MedicalRecord
      @relation(fields: [medicalRecordId], references: [id])
}
```

Create the enum.

```prisma
enum Severity {

  LOW

  MEDIUM

  HIGH

  CRITICAL
}
```

# Understanding Every Field

## id

```prisma
id String @id @default(uuid())
```

Unique identifier for every diagnosis.

Each diagnosis is stored independently.

---

## medicalRecordId

```prisma
medicalRecordId String
```

Links the diagnosis to the consultation where it was made.

A diagnosis cannot exist without a medical record.

---

## diagnosisName

```prisma
diagnosisName String
```

The doctor's conclusion.

Examples

```
Hypertension

Migraine

Diabetes

Viral Fever
```

---

## description

```prisma
description String?
```

Additional explanation.

Example

```
Stage 1 hypertension with elevated blood pressure.
```

---

## severity

```prisma
severity Severity
```

Represents the seriousness of the diagnosis.

Possible values

```
LOW

MEDIUM

HIGH

CRITICAL
```

Using an enum keeps values consistent.

---

## createdAt

Stores when the diagnosis was recorded.

---

## updatedAt

Automatically updates whenever the diagnosis changes.

---

## medicalRecord

```prisma
medicalRecord MedicalRecord
```

Creates the relationship between Diagnosis and Medical Record.

---

# Relationship

```
Medical Record (1)

        │

        ▼

Diagnosis (Many)
```

Every diagnosis belongs to one consultation.

A consultation may contain multiple diagnoses.

---

# PostgreSQL Representation

Conceptually

```sql
CREATE TABLE diagnoses (

    id UUID PRIMARY KEY,

    medical_record_id UUID NOT NULL,

    diagnosis_name TEXT NOT NULL,

    description TEXT,

    severity TEXT NOT NULL,

    created_at TIMESTAMP,

    updated_at TIMESTAMP,

    FOREIGN KEY (medical_record_id)

        REFERENCES medical_records(id)

);
```

# Why This Design?

A diagnosis is the doctor's medical conclusion.

It is different from:

- Consultation notes
- Medicines
- Follow-up instructions

Keeping diagnoses in a separate table allows:

- Multiple diagnoses per consultation.
- Better reporting.
- Easier searching.
- Future ICD code support.

# Business Flow

```
Patient Consultation

↓

Doctor examines patient

↓

Medical Record created

↓

Diagnosis added

↓

Prescription generated
```

Diagnosis comes after the consultation but before prescribing medication.

---

# Common Mistakes

### Saving Diagnosis Inside MedicalRecord

Medical records document the consultation.

Diagnoses document medical conclusions.

Keep them separate.

---

### Allowing Only One Diagnosis

Patients often have multiple conditions.

The schema should support multiple diagnoses.

---

### Using Free Text Severity

Avoid values such as

```
Very High

Medium+

Critical!!

```

Use enums.

---

# Senior Engineer Notes

Before approving this schema verify:

- Can one consultation have multiple diagnoses?

Yes.

- Can one diagnosis exist without a medical record?

No.

- Is diagnosis separate from prescription?

Yes.

Diagnosis identifies the condition.

Prescription identifies the treatment.

# Summary

The Diagnosis entity stores the doctor's medical conclusions for a consultation.

Each diagnosis belongs to one medical record, while a medical record may contain multiple diagnoses.

The next chapter introduces **Prescription**, where the doctor prescribes medicines based on these diagnoses.
