# 08.11 - Logout and Session Management

> Logging out is more than simply deleting a token on the client. In a production application, logout means invalidating the user's session so that stolen or previously issued Refresh Tokens can no longer be used. This chapter completes the authentication lifecycle by implementing secure session management.

# The Problem

Suppose a doctor logs into the application.

```
Laptop

↓

Refresh Token A
```

Later,

the doctor logs in on a mobile phone.

```
Mobile

↓

Refresh Token B
```

Both sessions are valid.

Now imagine the laptop is stolen.

Deleting the JWT from the browser is **not enough**.

The server must invalidate that Refresh Token.


# Why Logout Is Server Side

Many beginners think

```
localStorage.removeItem("token")
```

means logout.

It does not.

The server still trusts the Refresh Token.

Someone possessing that Refresh Token can still generate new Access Tokens.

Proper logout removes trust from the server.

# Repository

Modify

```
refresh-token.repository.ts
```

```ts
import { prisma } from '../../database/prisma'

export class RefreshTokenRepository {
  async delete(token: string) {
    return prisma.refreshToken.delete({
      where: {
        token
      }
    })
  }

  async deleteAll(userId: string) {
    return prisma.refreshToken.deleteMany({
      where: {
        userId
      }
    })
  }
}
```

---

# Why Repository?

The Repository only communicates with PostgreSQL.

Responsibilities

- Delete one Refresh Token
- Delete all Refresh Tokens

Business decisions belong to the Service.

---

# Service

Modify

```
auth.service.ts
```

```ts
import { RefreshTokenRepository } from "./refresh-token.repository";
import { UnauthorizedException } from "../../errors/UnauthorizedException";

const refreshRepository =
    new RefreshTokenRepository();

async logout(refreshToken: string) {

    const session =
        await refreshRepository.findByToken(refreshToken);

    if (!session) {

        throw new UnauthorizedException(
            "Session not found."
        );
    }

    await refreshRepository.delete(refreshToken);
}
```

---

# Why Validate First?

Suppose the client sends

```
Random Token
```

The backend should not silently ignore it.

Instead,

confirm the session exists before deleting it.

# Logout All Devices

```ts
async logoutAll(userId: string) {

    await refreshRepository.deleteAll(userId);
}
```

One query removes

```
Laptop

Phone

Tablet

Desktop
```

Every active session is revoked.

```ts
export async function logout(
  req: Request,

  res: Response
) {
  const { refreshToken } = req.body

  await service.logout(refreshToken)

  return res.status(200).json({

}
```

# Logout All Controller

```ts
export async function logoutAll(
```

# What Happens After Logout?

The Access Token may still be valid for a few minutes.

However,

once it expires,

the client attempts

```
POST /auth/refresh
```

The Refresh Token

no longer exists.

The backend returns

```
401 Unauthorized
```

The user must log in again.

---

# Session Management

Every Refresh Token represents

one active session.

Example

| Device | Status     |
| ------ | ---------- |
| Laptop | Active     |
| Phone  | Active     |
| Tablet | Logged Out |

Managing sessions becomes as simple as managing Refresh Tokens.

---

# Cleaning Expired Sessions

Expired Refresh Tokens should not remain forever.

Create a scheduled cleanup job.

Example

```
Every Day

↓

Delete Expired Refresh Tokens
```

SQL

```sql
DELETE FROM RefreshToken
WHERE expiresAt < NOW();
```

This keeps the database small and improves performance.

---

# Production Notes

Always

- Store Refresh Tokens in database.
- Delete tokens during logout.
- Support logout from all devices.
- Expire old sessions automatically.
- Never trust deleted sessions.
- Keep Access Tokens short-lived.

# Common Mistakes

## Client Only Logout

Wrong

```
Delete Token

↓

Finished
```

Correct

```
Delete Token

↓

Delete Database Session

↓

Logout
```

---

## Deleting Access Tokens Only

Refresh Tokens remain valid.

The user can still obtain new Access Tokens.

---

## Forgetting Session Cleanup

Inactive sessions accumulate over time.

Always remove expired Refresh Tokens.

# Summary

Logout is not simply removing tokens from the client.

A production backend must invalidate server-side sessions by revoking Refresh Tokens. This enables secure logout, multi-device session management, logout from all devices, and proper session lifecycle management.

By treating every Refresh Token as a session, the backend gains complete control over user authentication while maintaining strong security and an excellent user experience.
