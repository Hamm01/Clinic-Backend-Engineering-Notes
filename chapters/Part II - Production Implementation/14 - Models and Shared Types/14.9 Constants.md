# 14.9 - Constants

## Overview

Constants represent fixed values that are reused throughout the application.

Unlike enums, constants are not used to represent business states. Instead, they store configuration values, application settings, limits, messages, and other values that should not be duplicated.

Using constants improves maintainability by keeping commonly used values in a single location.

---

## Why Constants Exist

Imagine the following code.

```ts
const cacheTime = 300
```

Later in another file.

```ts
setTimeout(job, 300)
```

And somewhere else.

```ts
const TOKEN_EXPIRY = 300
```

What does **300** represent?

Nobody knows.

Instead of repeating magic numbers and strings, we define them once as constants.

---

## Implementation

In our Clinic Backend, constants are stored separately.

```text
src/

models/

└── constants/

    ├── api.constants.ts

    ├── auth.constants.ts

    ├── cache.constants.ts

    └── pagination.constants.ts
```

Each file groups constants related to one feature.

---

## Example

Authentication Constants

```ts
export const AUTH_CONSTANTS = {
  ACCESS_TOKEN_EXPIRY: '15m',

  REFRESH_TOKEN_EXPIRY: '7d',

  PASSWORD_MIN_LENGTH: 8
} as const
```

Usage

```ts
if (password.length < AUTH_CONSTANTS.PASSWORD_MIN_LENGTH) {
  throw new Error('Password is too short')
}
```

---

## Another Example

Pagination Constants

```ts
export const PAGINATION = {
  DEFAULT_PAGE: 1,

  DEFAULT_LIMIT: 10,

  MAX_LIMIT: 100
} as const
```

Usage

```ts
const page = PAGINATION.DEFAULT_PAGE

const limit = PAGINATION.DEFAULT_LIMIT
```

---

## API Route Constants

Some applications also centralize API routes.

```ts
export const API_ROUTES = {
  PATIENTS: '/patients',

  DOCTORS: '/doctors',

  APPOINTMENTS: '/appointments'
} as const
```

This avoids repeating route strings across the application.

---

## Request Lifecycle

Constants are not tied to any specific layer.

They can be used wherever required.

```text
Controller

↓

Service

↓

Repository

↓

Constants
```

Any layer can import and use a constant.

---

## Constants vs Enums

Enums

```ts
AppointmentStatus.COMPLETED
```

represent business values.

Constants

```ts
PAGINATION.DEFAULT_LIMIT
```

represent reusable application values.

---

## Best Practices

- Group related constants together.
- Use descriptive names.
- Replace magic numbers with constants.
- Replace repeated strings with constants.
- Keep constants immutable using `as const`.

---

## Common Mistakes

❌ Hardcoding numbers throughout the project.

❌ Repeating string literals.

❌ Creating one massive constants file.

❌ Using constants where enums should be used.

❌ Modifying constants during runtime.

---

## Summary

Constants provide a centralized location for reusable application values.

They improve readability, reduce duplication, and make configuration changes easier by keeping common values in one place.
